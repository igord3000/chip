# Chip Agent — Техническая документация

## Описание проекта

**Chip** — минимальный AI coding-агент с GUI, работающий локально через Ollama или через облачные провайдеры (OpenRouter, HuggingFace, OpenAI).

## Текущая архитектура

### UniversalAgent (основной)
```
Запрос → UniversalAgent → LLM (сам решает) → Инструменты → Ответ
```

### Ключевые файлы
1. `universal_agent.py` — основной агент, LLM сам решает что делать
2. `tools/` — инструменты (bash, файлы, интернет)
3. `ui/textual_app.py` — GUI интерфейс
4. `ui/settings.py` — настройки с выбором моделей

## Известные проблемы

1. **Контекст не сохраняется** между разными запросами в GUI
2. **LLM (qwen3:4b)** не всегда понимает уточняющие вопросы
3. **Нет streaming** — ответ показывается целиком после генерации
4. **Нет error recovery** при падении LLM

## Рекомендации по улучшению (от LLM)

### 1. Контекстный менеджер
```
ImprovedAgent → ContextManager → StreamingLLM → Tools → Answer
```
- `ContextManager` сохраняет диалог и обрабатывает уточнения
- Хранит только критически важные данные (последние 3-5 ответов)
- Использует векторные индексы для долгосрочного хранения

### 2. Streaming ответов
- Генерация ответа по частям в реальном времени
- Индикатор загрузки: "Генерирую ответ..."
- Пользователь видит текст по мере появления

### 3. Улучшение уточнений
- Автоматическое преобразование "а у диллеров?" → "цены у дилеров"
- Цепочка уточнений (retrieval-augmented QA)
- Тесты на разные типы уточнений

### 4. Архитектура
```
┌─────────────────────────────────────────────────────────┐
│                    User Input                           │
└─────────────────────────┬───────────────────────────────┘
                          │
                          ▼

│  • Сохраняет историю (последние 5 сообщений)          │
```

## Как продолжить разработку

### Шаг 1: Исправить контекст
Файл: `chip/universal_agent.py`
- Добавить сохранение `self.messages` между вызовами `execute()`
- Передавать историю из GUI в агент

### Шаг 2: Добавить streaming
Файл: `chip/llm.py`
- Использовать `stream=True` в запросе к LLM
- Показывать ответ по мере генерации в GUI

### Шаг 3: Улучшить промпты
Файл: `chip/universal_agent.py`
- Добавить больше примеров в system_prompt
- Улучшить обработку уточняющих вопросов

### Шаг 4: Тестирование
Файл: `tests/test_queries.py`
- Добавить больше тестовых запросов
- Проверить все типы инструментов

## Конфигурация

### ~/.chip/config.json
```json
{
  "model": "qwen3:4b",
  "base_url": "http://localhost:11434/v1",
  "api_key": "ollama",
  "max_tokens": 4096
}
```

### Переменные окружения
- `LLM_BASE_URL` — URL LLM сервера
- `LLM_API_KEY` — API ключ
- `LLM_MODEL` — модель
- `CONTEXT_MAX_TOKENS` — максимальный контекст
